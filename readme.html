YIV-Lib

High-Quality Image Library for C++ Apps

Description

YIV-Lib (Yaznbook Image Viewer Library) is a C++ library for loading, displaying, and processing high-resolution images. It supports all popular formats such as PNG, JPEG, BMP, GIF, TIFF, WebP, HEIF and provides features like filters, scaling, rotation, thumbnails, and thread-safe image lists.

Features

  • Load single or multiple images
  • High-resolution image support
  • Rotate & scale images
  • Apply filters: grayscale, invert, brightness, contrast
  • Generate thumbnails
  • Partial image loading (lazy)
  • Alpha channel detection
  • Thread-safe ImageList operations
  • Format conversion / save (PNG, JPEG, BMP, TGA)
  • Metadata/EXIF support stub

Requirements

  • C++17 compatible compiler
  • stb_image.h and stb_image_write.h (included or installed)
  • Optional: <thread> for multithreading support

Installation

1. Clone Repository

git clone https://github.com/YazanDev/yiv-lib.git
cd yiv-lib

2. Add to Your Project

Option A: Include Header + Source

  • Copy yiv-lib.h and yiv-lib.cpp into your project
  • Include in your code: #include "yiv-lib.h"

Option B: Build as Static/Shared Library

cmake_minimum_required(VERSION 3.10)
project(yiv-lib)

set(CMAKE_CXX_STANDARD 17)

add_library(yiv-lib STATIC yiv-lib.cpp)
target_include_directories(yiv-lib PUBLIC ${CMAKE_CURRENT_SOURCE_DIR})

Build:

mkdir build && cd build
cmake ..
make

Usage Example

#include <iostream>
#include "yiv-lib.h"

int main() {
    yiv::Image img;
    if (!img.loadFromFile("example.png")) return 1;

    img.applyFilter(yiv::FilterType::Grayscale);
    img.rotateClockwise();
    img.saveAs("example_out.png", yiv::ImageFormat::PNG);

    yiv::ImageList list;
    list.add(std::make_shared<yiv::Image>(img));
    std::cout << "ImageList contains " << list.count() << " image(s)." << std::endl;
    return 0;
}

Building Your Program with YIV-Lib

g++ main.cpp yiv-lib.cpp -o MyApp -std=c++17 -lpthread
./MyApp

Notes

  • Thread-safety implemented via std::mutex for ImageList
  • Large images may require more memory; use loadPartial or thumbnails
  • EXIF metadata parsing can be added later using libraries like libexif
